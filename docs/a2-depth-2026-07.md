# A2 纵深报告：18 集全适配与接入成本（2026-07）

**范围**：把已适配数据集从 12 扩到 **30**（三批 18 新集），每集走完全闸门（fetch+锁定 /
mapping.yaml 源字段全覆盖 / 薄适配器 / 集成层真实切片测试 / 许可阶梯黄金 / 无损往返 /
双仓 sha256 一致）。目的：以**接入成本**实证 UEP 的适配面到底有多厚、算子词汇表是否成熟、
撞到边界时协议能否演进。

诚实边界先说：这是**桌面适配 + 真实切片（各 ≥100 条，QuixBugs 全集 40）** 的工程实证，
不含模型出分；数据本体不入库（仅 sha256 锁定，复现见 `scripts/fetch_slices.py`）。

## 二、算子演进（7 个新算子/增强）

每个都对应一种**真实结构新形态**，非臆造：

| 算子 | 触发集 | 应对的结构 |
|------|--------|-----------|
| `text_match_from_boxed` | MATH | 最终答案裹在自由文本解答的 LaTeX `\boxed{}` 内（大括号配平，容嵌套） |
| `text_match_from_number` | MGSM | 答案是裸数值字段（非字符串/列表） |
| `execution_from_assertion_list` | MBPP | 测试是断言字符串列表（非单个 test_code + entry_point） |
| `retrieval_from_inline_passages` | MIRACL-zh | 文档以正/负段落**文本内联**给出（非引用式语料 + doc-id qrels） |
| `choice_match_from_bool` | BeaverTails | 答案是布尔标签（二元分类，配 const 固定二选项） |
| `options_from_fields.id_style`（增强） | MedMCQA | 非字母字段名（opa–opd）需位置字母 id |
| `choice_match_from_index.from_list`（增强） | GAOKAO | 正解是下标列表（多选） |

均自带 `invert()` 满足无损往返；增强均向后兼容（既有集测试未破）。

## 三、协议演进：撞到边界 → §8 提案

两集的判分形态现有 Verifier 表达不了，走 `custom` 受控逃生舱 + 正式 §8 演进提案
（同 phase-2.2 patch 字段演进的路径），这是**协议自演进循环**的实证：

- **成对偏好**（RewardBench）→ [`docs/proposals/2026-07-pairwise-preference.md`](proposals/2026-07-pairwise-preference.md)：提议 `preference` 原型 + Verifier（preferred/margin/ties/symmetric）。
- **指令跟随**（IFEval）→ [`docs/proposals/2026-07-instruction-following.md`](proposals/2026-07-instruction-following.md)：提议 `instruction_following` Verifier（命名判据表 + kwargs，可挂 qa 之上）。

两者的普查归属不同，如实区分：**RewardBench** 正是 A2 普查显式列入的分类学缺口（"成对偏好"簇）；
**IFEval** 普查桌面判为 `qa`/`partial`（备注留了"verifiable-reward 缺口"对冲），但**实测适配揭示
它需要一个新 `instruction_following` Verifier**——即真实适配比桌面 schema 普查更深地暴露了这个缺口
（覆盖地图 IFEval 行已注明此精化）。custom 条目一致性检查按逃生舱跳过（不判失败），完整判分谱存
payload，等提案落地。

## 四、诚实分类

- **full**（自足适配到五原型）：大多数。
- **partial** 1 集：DROP——passage（阅读材料）进 metadata 而非 task.question，因 qa 无一等"上下文"槽（亦为演进提示）。
- **custom** 2 集：RewardBench / IFEval（见上）。
- **无 golden** 的 6 集，原因分三类如实标注：custom 无试金石渲染（RewardBench/IFEval）、
  许可从严（BeaverTails NC / MIRACL-zh 未声明）、体量过大（HumanEval+ test 8–80KB）。
  所有集的**无损往返在全量真实切片上受检**，黄金缺席不影响正确性闸门。

## 五、全量转换规模

整 split（非 100 条切片）过适配器，坐实"万级"规模与转换吞吐（`scripts/fullscale_convert.py`，
数据本体不入库，摘要 `outputs/fullscale-2026-07.json`）：

| 集 | split | 全量条数 | 转换耗时 | 吞吐 | 无损抽验(200) |
|----|-------|---------|---------|------|--------------|
| MMLU-Pro | test | **12,032** | 0.35s | **34,186/s** | 0 差异 |
| MedMCQA | validation | **4,183** | 0.04s | 95,460/s | 0 差异 |
| **合计** | | **16,215** | | | 0 差异 |

**结论**：整 split 全量转换坐实"万级"（单 MMLU-Pro test 即 1.2 万条）。**关键发现**：
UEP 转换本身是**可忽略成本**（1.2 万条 0.35s ≈ 3.4 万条/秒）——全链瓶颈完全在**源数据
下载**（本轮 MMLU-Pro 抓取 481s，受 datasets-server 限流），与协议转换无关。即"UEP 是否
扛得住万级"对转换层根本不是问题：协议每条只加 ~微秒级开销，规模成本全在 UEP 之外的取数。
无损性在全量的抽验（每集首 200 条）0 差异，与切片上的全量往返一致。

## 六、覆盖广度

原型：choices / qa / retrieval（**引用式 + 内联**两条语料路径） / code（**Python + Java**两语言） / custom。
语言：中文 5 集（C-Eval / T2Ranking / GAOKAO / MGSM / MIRACL-zh）+ 英文主体。
域：通识 / 医学 / 数学 / 金融 / 数据科学 / 代码 / **安全** / 偏好 / 指令跟随。

## 结论

18 集全适配的接入成本表坐实：UEP 的算子词汇表成熟（半数 0 新算子），撞到真结构新形态时
小算子（类体 ~22–40 行，含 helper 的 boxed 约 53 行）即可容纳，撞到判分范式边界时
custom 逃生舱 + §8 提案让协议可演进。
适配面覆盖五原型、双语料路径、双语言、含 census 前沿缺口（偏好/指令/安全）。
